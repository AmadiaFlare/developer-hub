An RPC node (also referred to as an observer node) is responsible for servicing application requests.
Unlike [validator nodes](/run-node/register-node-as-validator), RPC nodes operate outside the network and do not influence consensus or block production.
Here you can find the minimum hardware requirements for running a node.

You have two options for setting up an RPC node, each with its own pros and cons:

1. [Setup on bare-metal](/run-node/from-source): More complex, but offers better performance and lower memory usage.

2. [Setup with Docker](/run-node/using-docker): Simpler, but results in lower performance and higher memory usage.

- **Disk speed:** 1200 MB/s read and 600 MB/s write, or higher
- **Network speed:** 40 Mbps, or higher

:::info

Adjust the compute requirements based on the number of requests per second you expect to serve.
The requirements above will be sufficient for most use cases, including handling RPC requests for an [FSP provider](/run-node/fsp-provider).

:::
